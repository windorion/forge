# Product Positioning

Document role: record how Forge should be explained, who it competes with,
where it is different, and what message should remain consistent across
product, website, and investor materials.

## Positioning Statement

Forge is the macOS-native AI software engineering workspace for developers who
want agents to complete real tasks while they stay in control of review,
architecture, and shipping.

## Competitive Frame

Forge is inspired by different strengths from existing products:

- Claude Code: strong agent loop and terminal workflow
- Codex: task execution, code modification, and reviewable output
- Cursor: mainstream AI coding adoption and developer ergonomics
- Raycast: macOS-native command surface and speed
- GitHub Desktop: approachable git workflows
- Linear: task-centered clarity and operational polish

Forge should not copy any one product. It should combine the missing layer
between them: a local, native workspace where agent work is visible and
reviewable.

## Core Differentiators

### 1. Not Editor-First

Most AI coding products start with the editor. Forge starts with the task.

### 2. Native macOS Surface

Forge can live in the menu bar, launcher, Finder, notifications, and desktop
workflow. It does not need the user to stay inside one IDE.

### 3. IDE-Agnostic

Forge should cooperate with Cursor, VS Code, Xcode, JetBrains IDEs, terminals,
and git repositories. The IDE is a tool; Forge is the orchestration layer.

### 4. Local-First Trust

Forge should keep repository context, memory, indexes, approvals, and task
history local by default.

### 5. Human-Review-Centered Autonomy

Forge should make agent work feel safe because it is observable, reversible,
and approval-based.

## Messaging Pillars

### For Developers

Give Forge a task. Review the result. Stay in control.

### For Teams

Make AI coding auditable, repeatable, and aligned with engineering workflow.

### For Enterprises

Local-first agent execution with explicit permissions, policy boundaries, and
reviewable changes.

## Naming

Current product name: Forge.

Former codename: Atlas.

Reasoning:

- Forge implies creating and shaping software.
- It supports product nouns such as Forge Runtime, Forge Mission, Forge
  Review, Forge Memory, and Forge Agent.
- It is less generic than Atlas and better aligned with software creation.

## Tagline Options

- Forge software with agents.
- The macOS workspace for agentic software engineering.
- Give agents the task. Keep humans in control.
- A local-first AI engineering workspace for Mac.

## Positioning Risks

- If Forge looks too much like an editor, users will compare it directly to
  Cursor and VS Code.
- If Forge looks too much like chat, users will not understand the workspace
  value.
- If Forge hides agent details, users will not trust it.
- If Forge ignores native macOS workflows, it loses its sharpest product wedge.
