# Forge — macOS App 设计交付包

> Forge 是 Windorion 旗下的 AI coding agent macOS 应用:自然语言下达任务 → agent 生成计划 → 人工批准 → agent 写码/自测/开 PR。
> 本包包含 **37 屏 macOS 应用界面设计稿**,供开发 agent(Claude Code / Codex 等)照稿实现。仅含 app 界面,不含 Windorion 网页端。

## 设计系统(必须严格遵守)

**风格**:developer-first 霓虹野兽派(neo-brutalist)。高对比、锐利、无渐变、无玻璃拟态、无圆角(应用自有 UI 内)。

- **底色**:`#f4f4f1`(纸灰白)· 卡片 `#fff` · 深色面板/终端 `#0a0a0a`
- **文字**:主 `#0a0a0a` · 次 `#2a2a26` · 弱 `#6a6a64` · 极弱 `#9a9a92`
- **强调色**:`#a674ff`(紫,CSS 变量 `--accent`,可替换)— 用于运行态、选中态、主按钮文字(黑底上)、进度条
- **语义色**:警告/等待 `#febc2e`(黄) · 危险 `#c0392b`(红) · 成功 `#28c840`(绿,仅小图标) · diff 绿 `#7ee787` / diff 红 `#ff8a80`(仅深色终端内)
- **边框**:统一 `1.5px solid #0a0a0a`;次级分隔 `#e2e1dc`;禁用/未选中 `#cccac2`
- **阴影**:硬阴影,无模糊 — 窗口 `10px 10px 0 rgba(10,10,10,0.85)`,卡片 `4-5px 4-5px 0 #0a0a0a`;hover 主按钮位移 1px + 阴影收缩(压下去的感觉)
- **字体**:UI 标签/数据/代码 = `JetBrains Mono`(Google Fonts);正文 = Helvetica Neue。小标签一律大写 + letter-spacing
- **圆角**:应用自有 UI 为 0;**唯一例外**是 macOS 系统渲染的部分(见下)

## 两类界面 — 关键区分

画布中标注「原生样式 / 原生」的屏,由 macOS 系统渲染,**样式不可定制**,稿内是系统规范的还原,开发时只提供数据/元数据:

- `8a` Dock 右键菜单(NSMenu,纯文本项)
- `9a` 系统通知横幅(UNUserNotificationCenter)
- `11a` Spotlight(Core Spotlight 索引元数据)
- `21a` 应用菜单命令(NSMenu)
- `35a` 桌面小组件(WidgetKit — 容器系统强制圆角,内容排版可自定义,稿内内容区仍用品牌语言)

其余全部为应用自有 UI(SwiftUI/AppKit 自绘),必须按稿实现野兽派样式。

## 产品原则(设计里反复出现,实现时不可破坏)

1. **计划闸门**:任何入口(主窗/聊天/⌘K/浮窗/CLI/模板)创建的任务,都必须经过计划审批才动代码
2. **绝不擅自决策**:决策点暂停提问(33a/34a),未回答的任务保持暂停,"nothing is guessed"
3. **最小权限**:GitHub 只要 repo:read + branch:write(仅 forge/* 分支)+ pr:open,无 merge/delete/admin
4. **checkpoint 每步落盘**:断网(29a)、崩溃(31a)、断电都能恢复,分支永不半写
5. **预算护栏**:$40 月上限,触顶暂停不中止;暂停等待时不产生模型调用费用
6. **API key 不经过服务器**:Keychain 本地存储,调用直连 provider

## 交互行为速查(稿内可点的都要实现)

- 选中态:紫底 `#a674ff` + 硬阴影 + 黑边;未选中:灰边 `#cccac2` 白底
- 切换控件:分段按钮(黑底白字为当前),开关(紫底右滑为开)
- 手风琴:▸/▾ 折叠展开(37a 成本步骤、FAQ 类)
- 进度条:方角、黑边、紫色填充、内部右边框分隔
- 终端/思考流:黑底 `#0a0a0a`、时间戳灰字、闪烁光标(`steps(1)` 动画)

## 给接手 agent 的建议

- 目标平台 macOS 14+,SwiftUI 优先;窗口尺寸稿内为 1:1 像素(主窗 1380×,设置 980×)
- `JetBrains Mono` 需随 app 打包(SIL OFL 许可)
- 所有数据均为示意假数据(任务 #128、$0.83 等),接真实数据时保持格式(等宽、单位、精度)
- 实现顺序建议:14a 主窗 → 1a/1b 任务创建+审批 → 10a diff 审查 → 设置组 → 系统集成 → 边缘状态
